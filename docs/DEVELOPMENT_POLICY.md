# 開発ポリシー (Development Policy)

## 1. サードパーティコードとライセンス
- **リポジトリへの同梱**: 著作権やライセンス上の問題を避けるため、サードパーティ製のライブラリやバイナリをリポジトリに直接含めないでください。
- **依存管理**: パッケージマネージャー (pip, npm) やビルドスクリプト (Docker, Make) を使用して依存関係を解決してください。
- **アセット取得**: JSライブラリ（例: `crypto-js`）などの静的アセットは、コミットせずにビルドプロセス中（例: Dockerfile内）にダウンロードするように構成してください。

## 2. デバッグモードと認証情報
- **デバッグログ**: トラブルシューティングのためにログ出力を詳細にする機能（デバッグモード）を実装する場合、APIキーやパスワードなどの認証情報がログに出力されないよう厳重に注意してください。

## 3. コード品質
- **Pythonバージョン**: Python 3.10以上で動作することを確認してください。
- **型ヒント**: コードの可読性と堅牢性を高めるため、Type Hintsの活用を推奨します。
- **コーディング規約**: PEP 8 に準拠してください。

## 4. ドキュメンテーション
- **英語の禁止（原則）**: ドキュメントは原則として日本語で記述し、読みやすさを維持してください。
- **更新**: 要件が変更された場合は `SPEC.md` を更新し、使用方法は `README.md` に最新情報を反映してください。
